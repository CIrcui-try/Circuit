# Testing Strategy

## Decision

Add UI/E2E testing now. Do not wait until the product is fully implemented.

Phase 03 exists specifically to create the test infrastructure before the visual flow editor becomes complex.

## Tooling

Use:

```text
Vitest for core logic
Playwright for UI/E2E flows
```

## E2E Target

Early E2E tests should target the React/Vite app rather than the fully packaged Tauri app.

The Tauri bridge must be mockable.

## Native Dialog Rule

Do not automate native macOS file dialogs directly.

Instead, repository selection must go through a bridge function:

```ts
openRepositoryDialog(): Promise<RepositoryPath | null>
```

In Playwright tests, mock this bridge and return a fixture path.

## Test Fixtures

Create test repositories under a fixture directory.

Example:

```text
fixtures/
└── repos/
    └── sample-repo/
        ├── .claude/
        │   └── skills/
        │       └── implement-feature/
        │           └── SKILL.md
        ├── .codex/
        │   └── skills/
        │       └── review-code/
        │           └── SKILL.md
        └── docs/
            └── ignored-skill/
                └── SKILL.md
```

The `docs/ignored-skill/SKILL.md` file exists to verify that Circuit does not scan arbitrary repository paths.

## Required Test Coverage by Phase

```text
Phase 03: app smoke test, repository list smoke test, mocked bridge setup
Phase 04: add skill as node, select node, connect nodes
Phase 05: save workflow, load workflow, restore graph
Phase 06: Start Circuit button, sequential mock execution, duplicate run prevention
Phase 07: running/success/failed visual states and run log
Phase 08: schema is agent-readable and preserves skill references
```

## Test ID Policy

Important UI elements must have stable test identifiers.

Examples:

```text
data-testid="repository-list"
data-testid="add-repository-button"
data-testid="skill-list"
data-testid="workflow-canvas"
data-testid="start-circuit-button"
data-testid="run-log"
data-testid="node-properties-panel"
```

Avoid selecting elements only by CSS class names generated by UI libraries.
