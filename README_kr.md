# Circuit

[English README](README.md)

#### Circuit은 에이전트 행동을 예측 가능하게 만들어주는 워크플로우 시각화 툴입니다.

Circuit은 Claude와 Codex 스킬을 시각적인 워크플로로 연결해 실행하는 데스크톱 앱입니다.

Circuit은 AI-native 프로젝트를 운영하면서 자주 느꼈던 불편함에서 시작했습니다. 에이전트에게 플래닝, 구현, 리뷰 같은 작업을 맡길수록 생산성은 좋아졌지만 직접 보고 있지 않은 동안 에이전트가 어떤 흐름으로 움직이고 있는지 파악하고 통제하기는 어려웠습니다. 프로젝트가 커질수록 어느 단계가 끝났고 어디서 개입해야 하는지 확인하기 위해 엄청난 양의 터미널 출력과 프롬프트를 되짚는 일이 점점 잦아졌습니다.

특히 플래닝, 구현, 리뷰, 커밋, 머지, 배포처럼 반복되는 작업을 에이전트에게 맡기다 보면 흐름은 점점 길어집니다. 자동화 자체는 강력하지만, 긴 프롬프트, 명령어 체인, TUI 세션 안에서 실행되기 시작하면 지금 어느 단계인지, 무엇이 끝났는지, 어디서 실패했는지 다시 파악하기가 어려워집니다.

Circuit은 이 문제를 가시적으로 다루기 위해 만들었습니다. 로컬 저장소에 있는 Claude와 Codex 스킬을 자동으로 읽어오고, 필요한 스킬을 캔버스 위에 블록처럼 배치한 뒤, 순서와 의존성을 연결해 하나의 루틴으로 실행할 수 있습니다. 에이전트가 어느 단계에 있는지, 어느 단계가 끝났는지, 어디서 실패했는지도 실행 로그와 캔버스를 통해 확인할 수 있습니다.

Circuit은 에이전트 자동화에 익숙한 고급 사용자만을 위한 도구가 아닙니다. 스킬과 루틴을 막 만들기 시작한 사용자, 매번 비슷한 작업을 반복하지만 아직 명령어 기반 자동화가 익숙하지 않은 사용자도 자신만의 워크플로를 만들고 실행할 수 있도록 돕는 것이 목표입니다.

## 왜 Circuit인가요?

에이전트를 사용하다 보면 다음과 같은 루틴이 자주 반복됩니다.

```text
plan → implement → review → commit → merge → deploy
```

하지만 실제 작업 흐름은 항상 이렇게 깔끔하게 고정되지 않습니다. 어떤 작업에서는 리뷰 후 다시 구현 단계로 돌아가야 하고, 어떤 작업에서는 커밋까지만 해둔 채 머지는 미뤄야 할 수도 있습니다. 큰 변경 사이에는 토큰 사용량을 확인하고 잠시 멈춰야 할 수 있고 컨텍스트가 길어졌다면 compact 단계를 넣어야 할 수도 있습니다.

문제는 이런 순서와 의존성이 계속 바뀐다는 점입니다. 텍스트로 된 명령 체인이나 터미널 히스토리만으로는 어떤 스킬이 어떤 순서로 실행되는지, 어디에 새 단계를 끼워 넣어야 하는지, 어느 지점에서 사람이 개입해야 하는지 쉽게 보이지 않습니다.

Circuit은 이 흐름을 캔버스 위에서 직접 다룹니다. 필요한 스킬을 블록처럼 가져오고, 순서를 바꾸고, 의존성을 연결하고, 새로운 단계를 끼워 넣을 수 있습니다. 리뷰를 커밋 앞에 둘지 뒤에 둘지 같은 변경도 명령어를 다시 작성하는 대신 노드를 옮기고 다시 연결하면 됩니다.

## 워크플로 화면

<!-- ![Claude와 Codex 스킬 사이드바, 연결된 워크플로 캔버스, 실행 중인 Run Log가 보이는 Circuit 워크스페이스.](docs/assets/readme/circuit-workflow-overview.png) -->

Circuit 워크스페이스는 로컬 저장소에서 발견한 Claude와 Codex 스킬을 한 화면에 모아 보여줍니다.

왼쪽에서는 사용할 수 있는 스킬을 확인하고, 중앙 캔버스에서는 스킬을 배치하고 연결합니다. 실행 중에는 Run Log를 통해 현재 진행 상황을 확인할 수 있습니다. 워크플로가 실패했을 때도 어느 단계까지 실행되었고 어디서 멈췄는지 확인하기 쉽습니다.

## 주요 기능

- 로컬 저장소를 등록해 프로젝트별 스킬과 워크플로를 관리합니다.
- Claude와 Codex 스킬을 함께 스캔합니다.
  - `.claude/skills/*/SKILL.md`
  - `.codex/skills/*/SKILL.md`
- Claude 스킬과 Codex 스킬을 같은 캔버스에 배치할 수 있습니다.
- 스킬을 블록처럼 배치하고 의존성 엣지로 연결할 수 있습니다.
- 저장소별 워크플로 초안을 저장하고 다시 불러올 수 있습니다.
- 시스템 스킬과 기본 스킬을 모든 워크플로에서 재사용할 수 있습니다.
- 수동으로 워크플로를 실행하고, 실행 로그를 앱 안에서 확인할 수 있습니다.
- 실행 중인 워크플로를 취소할 수 있습니다.
- 루프가 있는 워크플로를 만들 수 있습니다.
- 순환 그래프는 실행 전에 경고를 표시한 뒤, 사용자가 명시적으로 선택하면 실행할 수 있습니다.
- Claude와 Codex provider adapter를 통해 하나의 런타임에 고정되지 않는 구조를 제공합니다.

## 스킬들을 루틴으로 만들기

Circuit의 핵심은 스킬 하나를 실행하는 것이 아니라, 여러 스킬을 하나의 루틴으로 묶는 것입니다.

예를 들어 다음과 같은 흐름을 만들 수 있습니다.

```text
planning → implementation → review → commit
```

작업 방식이 바뀌면 루틴도 쉽게 바꿀 수 있습니다.

```text
planning → implementation → commit → review
```

중간에 점검 단계가 필요하다면 새 스킬을 끼워 넣으면 됩니다.

```text
planning → check-token → implementation → review → wrap-up
```

컨텍스트가 길어졌을 때는 `compact` 스킬을 넣을 수 있고, 작업 이후 정리가 필요하다면 `wrap-up` 스킬을 붙일 수 있습니다. 토큰 사용량을 확인하고 싶다면 `check-token` 같은 스킬을 만들어 큰 단계 사이에 배치할 수 있습니다.

이런 흐름은 명령어 목록으로도 만들 수 있지만, Circuit은 그것을 눈에 보이는 그래프로 다룹니다. 스킬의 순서와 의존성이 바뀔 때마다 전체 절차를 다시 떠올릴 필요 없이, 노드와 엣지를 수정하면 됩니다.

## Claude와 Codex를 함께 사용하기

많은 프로젝트에는 이미 여러 종류의 에이전트 자동화가 섞여 있습니다. 어떤 루틴은 Claude 스킬로 만들어져 있고, 어떤 루틴은 Codex 스킬로 관리될 수 있습니다.

Circuit은 이 둘을 경쟁하는 도구로 보지 않습니다. 로컬 프로젝트가 가진 서로 다른 실행 능력으로 보고, 하나의 워크플로 안에서 함께 다룰 수 있게 합니다.

실제 스킬 파일은 계속 저장소 안에 남아 있습니다. Circuit은 그 파일을 다른 곳으로 옮기거나 별도의 포맷으로 강제하지 않습니다. 대신 저장소에 있는 스킬을 읽어와 캔버스 위에 보여주고, 어떤 순서와 의존성으로 실행할지 정의하는 시각 레이어를 제공합니다.

## 루프와 반복 흐름

모든 워크플로가 직선으로 끝나지는 않습니다. 어떤 루틴은 반복이 필요합니다. 예를 들어 실패한 작업을 다시 검토하거나, 일정 조건을 만족할 때까지 점검 단계를 반복하고 싶을 수 있습니다.

Circuit은 루프를 무조건 막지 않습니다. 순환이 있는 그래프는 무한 실행될 수 있으므로 시작 전에 경고를 표시하지만, 사용자가 의도한 반복 흐름이라면 실행할 수 있도록 둡니다.

중요한 점은 루프가 숨겨져 있지 않다는 것입니다. 캔버스 위에 반복 흐름이 그대로 드러나기 때문에, 어떤 스킬이 다시 호출되는지 확인하고 조정할 수 있습니다.

## 로컬 우선 모델

Circuit은 local-first 앱입니다. 스킬과 워크플로는 사용자의 로컬 저장소를 기준으로 동작합니다.

- 스킬 디스커버리는 선택한 저장소의 파일을 읽어 수행합니다.
- Claude와 Codex 스킬 정의는 기존 저장소 구조를 그대로 사용합니다.
- 워크플로 실행은 Tauri backend bridge를 통해 처리됩니다.
- Claude와 Codex 실행은 provider adapter 인터페이스를 통해 연결됩니다.
- 실행 출력은 앱 안의 로그 패널로 스트리밍됩니다.
- 안전에 민감한 런타임 동작은 원격 서비스가 아니라 로컬 환경에서 다뤄집니다.

현재 Circuit은 명시적인 수동 실행을 중심으로 동작합니다. 파일 변경을 자동 트리거로 삼거나, git remote로 push하거나, 배포를 수행하거나, 임의의 shell-command 노드를 실행하지 않습니다.

## 현재 상태

Circuit은 활발히 개발 중입니다. 현재 앱에는 다음 기능들이 구현되어 있습니다.

- 저장소 등록과 제거
- Claude와 Codex 스킬 디렉터리 스캔
- 번들된 시스템 스킬과 기본 스킬 디스커버리
- 저장소 목록의 provider count badge
- 워크플로 캔버스 편집
- 워크플로 초안 저장과 불러오기
- 수동 워크플로 실행
- 실행 상태와 실행 로그 표시
- 실행 중인 워크플로 취소
- 루프 감지와 명시적 실행 확인
- Claude와 Codex adapter 구현

현재 알려진 제한 사항은 다음과 같습니다.

- 협업 또는 공유 원격 워크스페이스는 아직 없습니다.
- 자동 배포나 git push 동작은 제공하지 않습니다.
- 사용자 설정 전역 스킬 디렉터리 디스커버리는 아직 없습니다.
- 임의 shell command 노드 타입은 없습니다.
- 런타임 표면은 앱이 발전하면서 계속 강화 중입니다.

## 개발

앱 코드는 `app/`에 있으며 React, Vite, Tauri, Vitest, Playwright를 사용합니다.

```sh
cd app
pnpm install
pnpm dev
```

자주 쓰는 확인 명령은 다음과 같습니다.

```sh
pnpm test:run
pnpm build
pnpm test:e2e
```
